# 改动总结 - 2026-03-10 (第二个会话)

## 问题排查和改进

### 1. ✅ 修复登录错误处理问题
**问题:** 输入错误密码或正确密码都提示网络错误
**原因:** 前端认证服务的错误处理不够精细
**解决方案:** 改进 `frontend/src/services/auth.ts` 的错误处理逻辑
  - 区分真正的网络错误和 API 返回的错误
  - TypeError 时提示"网络错误，请检查服务器地址"
  - 其他错误时提示"登录失败，请稍后重试"
  - 成功解析服务器错误消息并显示

**测试结果:** ✅
```
正确密码: ✓ 返回 200 和 JWT token
错误密码: ✓ 返回 401 和错误信息 "Invalid username or password"
空密码: ✓ 返回 400 和错误信息 "Username and password are required"
```

### 2. ✅ 替换密码框图标为专业眼睛图标
**问题:** 使用 emoji 表情 (👁️ 🙈) 不够专业
**解决方案:** 
  - 在 `Login.tsx` 中导入 `Eye` 和 `EyeOff` 图标
  - 替换所有密码显示/隐藏按钮的 emoji 为 lucide-react 图标
  - 添加 ARIA 标签提高无障碍访问

**文件修改:**
  - `frontend/src/pages/Login.tsx` - 添加导入和替换图标
  - `frontend/src/pages/ForceChangePassword.tsx` - 两个密码框都替换

### 3. ✅ 移除前端显示的默认账户信息
**问题:** Login 页面显示"默认账户: admin / admin"不安全
**解决方案:**
  - 删除 Login.tsx 页脚中的默认账户提示
  - 改为提示"初始凭证在服务器启动时输出到控制台"
  - 将初始密码信息移到服务器日志和 README

**文件修改:**
  - `frontend/src/pages/Login.tsx` - 移除默认账户显示

### 4. ✅ 在服务器启动时显示初始凭证
**问题:** 用户不知道初始账户密码
**解决方案:**
