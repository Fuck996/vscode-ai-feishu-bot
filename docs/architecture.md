# 架构设计

## 系统架构

```
┌─────────────────────────────────────────────────────────────────┐
│                        AI 工作流程                                │
│  (GitHub Actions, GitLab CI/CD, 本地脚本等)                      │
└────────────────────────────┬────────────────────────────────────┘
                             │
                             │ Webhook 请求
                             ▼
┌─────────────────────────────────────────────────────────────────┐
│                      后端服务 (Node.js)                          │
│  ┌──────────────────────────────────────────────────────────┐  │
│  │  Express 服务器                                           │  │
│  │  - /api/notify          (接收通知)                        │  │
│  │  - /api/notifications   (查询历史)                        │  │
│  │  - /api/stats           (统计数据)                        │  │
│  │  - /api/health          (健康检查)                        │  │
│  └──────────────────────────────────────────────────────────┘  │
│  ┌──────────────────────────────────────────────────────────┐  │
│  │  服务层                                                   │  │
│  │  - Webhook 验证和处理                                    │  │
│  │  - 飞书 API 调用                                          │  │
│  │  - 消息格式化                                            │  │
│  │  - 错误重试                                              │  │
│  └──────────────────────────────────────────────────────────┘  │
│  ┌──────────────────────────────────────────────────────────┐  │
│  │  数据层                                                   │  │
│  │  - SQLite 数据库                                         │  │
│  │  - 通知历史记录                                          │  │
│  │  - API 密钥管理                                          │  │
│  └──────────────────────────────────────────────────────────┘  │
└────────────────────────────┬────────────────────────────────────┘
         │                   │
         │                   │
    ┌────▼────┐         ┌────▼────┐
    │  飞书   │         │  前端   │
    │ Webhook │         │ React   │
    └────┬────┘         └────┬────┘
         │                   │
         ▼                   ▼
    ┌────────────┐      ┌────────────┐
    │  飞书机器人 │      │ 管理面板   │
    │  收到通知   │      │ Dashboard  │
    └────────────┘      └────────────┘
```

## 数据流

### 1. 通知流程

```
外部系统 → HTTP POST /api/notify → 验证请求 → 保存数据库 → 发送飞书 → 返回结果
```

### 2. 查询流程

```
前端 → HTTP GET /api/notifications → 数据库查询 → 返回结果 → 前端渲染
```

## 核心模块

### 后端 (Backend)

**server.ts** - 主服务器
- Express 应用初始化
- 中间件配置 (CORS, 限流, 日志等)
- 路由注册
- 数据库初始化

**webhook.ts** - 接口处理
- POST /api/notify - 接收并处理通知
- GET /api/notifications - 查询历史  
- GET /api/stats - 获取统计数据
- POST /api/webhooks/test - 测试连接

**feishu.ts** - 飞书集成
- 发送富文本消息到飞书
- 消息格式化
- 重试机制
- 错误处理

**database.ts** - 数据持久化
- SQLite 数据库操作
- 通知表管理
- API 密钥验证
- 统计函数

**config.ts** - 配置管理
- 环境变量加载
- 配置验证
- 默认值设置

**logger.ts** - 日志系统
- Pino 日志库
- 结构化日志
- 多级别日志

### 前端 (Frontend)

**App.tsx** - 应用主体
- 路由配置
- 导航栏
- 布局管理

**Dashboard.tsx** - 仪表板
- 统计数据显示
- 图表组件
- 实时更新

**History.tsx** - 历史记录
- 通知列表
- 筛选功能
- 分页

**Settings.tsx** - 设置页面
- 服务器 URL 配置
- 连接测试
- 部署指南

### VSCode 扩展 (Extension)

**extension.ts** - 扩展入口
- 命令注册
- 事件监听
- 配置管理

**feishuNotifier.ts** - 通知器类
- 请求封装
- 重试机制
- 便利方法

## 部署架构

### 本地部署

```
docker-compose up
│
├── Backend (Node.js)
│   └── SQLite 数据库
│
└── Frontend (Nginx)
    └── React SPA
```

### 云端部署 (Railway 示例)

```
Railway App
├── Backend Service
│   ├── Node.js 容器
│   └── SQLite 数据卷
│
└── Frontend Service
    ├── Nginx 容器
    └── 正向开放 URL
```

## 安全性设计

1. **认证**
   - JWT 令牌验证
   - API 密钥管理

2. **输入验证**
   - Joi schema 验证
   - 请求体大小限制
   - 参数类型检查

3. **速率限制**
   - express-rate-limit 中间件
   - 单 IP 限流

4. **数据加密**
   - HTTPS (生产环境)
   - 敏感信息从环境变量读取

5. **错误处理**
   - 统一错误处理
   - 信息脱敏
   - 详细日志
