**版本：** v1.0.0 | **更新时间：** 2026-03-12 | **内容：** 密码找回功能设计（飞书机器人辅助验证）

## 📋 功能概述

### 背景
- 当前用户信息页面收集邮箱，但无实际用途
- 需要移除邮箱收集功能
- 新增密码找回功能，通过飞书机器人进行身份验证

### 核心流程
```
用户输入用户名 
  ↓
系统生成临时验证码 + 有效期（10分钟）
  ↓
通过飞书机器人发送验证码到用户
  ↓
用户在飞书收到验证码
  ↓
用户在密码找回页面输入验证码
  ↓
系统验证码有效性后，允许设置新密码
  ↓
密码更新成功，回到登录页
```

## 🔌 后端 API 设计

### 新增端点

#### 1. 请求验证码
```
POST /api/auth/forgot-password/request-code

请求体:
{
  "username": "admin"
}

响应（成功）:
{
  "success": true,
  "message": "验证码已通过飞书机器人发送，请查收",
  "expiresIn": 600  // 秒，10分钟
}

响应（失败）:
{
  "success": false,
  "error": "用户不存在"  // 或 "发送失败，请稍后重试"
}
```

#### 2. 验证码校验
```
POST /api/auth/forgot-password/verify-code

请求体:
{
  "username": "admin",
  "code": "123456"
}

响应（成功）:
{
  "success": true,
  "token": "temporary_reset_token_xxx",  // 一次性 Token，用于重置密码
  "expiresIn": 3600  // 秒，1小时
}

响应（失败）:
{
  "success": false,
  "error": "验证码错误或已过期"
}
```

#### 3. 重置密码
```
POST /api/auth/forgot-password/reset

请求头:
{
  "X-Reset-Token": "temporary_reset_token_xxx"
}

请求体:
{
  "newPassword": "NewPassword@123"
}

响应（成功）:
{
  "success": true,
  "message": "密码已重置，请重新登录"
}

响应（失败）:
{
  "success": false,
  "error": "Token 无效或已过期"  // 或 "密码格式错误"
}
```

## 💾 数据存储

### 验证码临时存储（内存或数据库）

```typescript
interface PasswordResetCode {
  username: string;
  code: string;  // 6位数字
  createdAt: number;  // 毫秒时间戳
  expiresAt: number;  // 毫秒时间戳（10分钟后）
  attempts: number;  // 失败尝试次数
  maxAttempts: 5;  // 最多尝试 5 次
}
```

### 重置令牌（临时存储）

```typescript
interface PasswordResetToken {
  username: string;
  token: string;  // UUID
  createdAt: number;
  expiresAt: number;  // 1小时后
}
```

## 🚀 实现步骤

### 后端实现

1. **在 auth.ts 路由中添加三个新端点**
   - `POST /forgot-password/request-code` - 生成验证码并发送
   - `POST /forgot-password/verify-code` - 验证码校验
   - `POST /forgot-password/reset` - 重置密码

2. **验证码生成与存储**
   - 使用 6 位随机数字
   - 10 分钟有效期
   - 防暴力破解：限制失败尝试次数（5次）和发送频率

3. **飞书通知集成**
   - 使用现有的 feishuService 发送验证码
   - 消息内容：新用户用户名 + 验证码 + 有效期

4. **临时令牌生成**
   - 验证码通过后生成一次性令牌
   - 用于后续的密码重置请求
   - 1小时有效期

### 前端实现

1. **添加找回密码页面** (`frontend/src/pages/ForgotPassword.tsx`)
   - 三步表单流程
   - 倒计时显示
   - 实时验证密码强度

2. **修改登录页面**
   - 添加"找回密码"链接
   - 使用 React Router 导航

3. **用户信息页面修改**
   - 移除邮箱收集字段
   - 修改为已验证的小显示文本（锁定）

## ✅ 安全考虑

1. **防暴力破解**
   - 限制单个用户的验证码发送频率（1分钟内最多发送1次）
   - 限制验证码失败尝试次数（5次后需要重新请求）
   - 限制密码重置失败尝试次数

2. **令牌安全**
   - 重置令牌一次性使用
   - 设定自动过期时间（1小时）
   - 使用 HTTPS 传输

3. **验证码传输**
   - 通过飞书机器人私信发送（不通过邮件或短信）
   - 验证码存储加密（bcrypt 或类似）

4. **日志记录**
   - 记录所有密码重置操作
   - 包含用户名、时间、IP地址

## 🎯 交互细节

### 倒计时显示
- 验证码有效期为 10 分钟
- 前端实时倒计时，当剩余时间 < 1 分钟时显示红色警告
- 到期自动清空表单

### 密码强度提示
```
强度等级：
🔴 弱 - 少于 8 字符或缺少必需字符
🟡 中 - 8+ 字符，包含大小写字母和数字但无特殊字符
🟢 强 - 8+ 字符，包含大小写字母、数字、特殊字符
```

### 错误提示
- 验证码错误：显示"验证码错误，您还有 4 次尝试机会"
- Token 过期：自动返回第一步，需要重新请求
