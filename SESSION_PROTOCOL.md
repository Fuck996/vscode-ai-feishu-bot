# 对话后约定执行操作指南

**创建时间:** 2026-03-10  
**目的:** 规范每次对话结束后必须执行的操作，确保项目状态一致

---

## 🎯 核心原则

每次对话结束时，应该确保：
1. ✅ 所有修改都已保存到文件
2. ✅ 关键文件位置清晰可追踪
3. ✅ 下一次对话能快速理解当前状态
4. ✅ 后续工作有明确的下一步

---

## 📋 对话后检查清单

### 类别 1: 文件操作确认

- [ ] **所有修改已保存** - 确认没有未保存的临时文件
- [ ] **文件位置正确** - 核实新建/移动的文件在预期位置
- [ ] **旧文件清理** - 删除重复或废弃的文件
- [ ] **目录结构更新** - 如有变化，更新相关参考文档

**执行方式:**
```bash
# 检查当前修改
git status

# 查看文件列表
ls -la docs/
ls -la backend/
ls -la frontend/
```

---

### 类别 2: 文档一致性检查

- [ ] **文档引用正确** - 所有文档链接指向正确的位置
- [ ] **版本号已更新** - 修改的文档版本号递增
- [ ] **目录结构文档同步** - README或导航文档已更新
- [ ] **相关文档交叉引用** - 关联文档互相链接

**执行方式:**
```bash
# 查找所有Markdown文件中的链接
grep -r "\[.*\](.*\.md)" --include="*.md"

# 检查版本号
grep -r "版本:" docs/ --include="*.md"
```

---

### 类别 3: 后续工作明确化

创建或更新 `当前任务总结` 部分，包含：

- [ ] **已完成工作** - 列出本次对话完成的所有任务
- [ ] **当前状态** - 系统现在处于什么状态
- [ ] **已知问题** - 记录任何发现的问题或限制
- [ ] **下一步任务** - 明确下一次对话应该做什么

**执行方式:**

在会话总结中记录：
```markdown
## 本次对话总结

### ✅ 已完成
- 任务1
- 任务2
- ...

### 🟡 当前状态
- 系统状态：[描述]
- 关键文件：[列出]
- 运行状态：[后端/前端状态]

### ❌ 已知问题
- 问题1：[描述]
- 问题2：[描述]

### 🚀 下一步
1. [优先级1] 任务A
2. [优先级2] 任务B
3. [优先级3] 任务C
```

---

### 类别 4: 代码质量检查

- [ ] **无语法错误** - 新写的代码没有明显错误
- [ ] **测试通过** - 如有测试，应该通过
- [ ] **依赖完整** - 新文件的依赖都已列出
- [ ] **注释清晰** - 重要代码有说明注释

**执行方式:**
```bash
# 检查TypeScript错误
cd frontend && npm run type-check
cd backend && npm run type-check

# 运行任何可用的测试
npm test
```

---

### 类别 5: 环境一致性确认

- [ ] **后端运行状态** - 本地后端能正常启动
- [ ] **前端运行状态** - 本地前端能正常启动
- [ ] **API连通性** - 后端和前端能正常通信
- [ ] **数据库状态** - 数据存储层运行正常

**执行方式:**
```bash
# 后端
cd backend
npm run dev

# 前端（新终端）
cd frontend
npm run dev

# 验证API
curl http://localhost:3000/api/health
```

## 🔄 三种对话模式的不同要求

### 模式 A: 设计/文档对话

**对话类型:** 创建设计文档、讨论架构、制定计划

**必须执行:**
1. ✅ 文档已创建到正确位置
2. ✅ 版本号已标记
3. ✅ STATUS_AND_ROADMAP.md已更新
4. ✅ 下一步任务明确

**检查命令:**
```bash
find docs/ -name "*.md" -type f -mmin -60  # 查看最近一小时修改的文件
head -5 [document].md   # 检查文档开头（版本号等）
```

---

### 模式 B: 功能实现对话

**对话类型:** 编写代码、实现功能、修复Bug

**必须执行:**
1. ✅ 代码已保存
2. ✅ 无编译错误
3. ✅ 本地环境可运行
4. ✅ 端口绑定正确（后端:3000, 前端:5175）
5. ✅ 下一步清晰

**检查命令:**
```bash
git status              # 查看修改的文件
npm run dev            # 验证能启动
curl localhost:3000    # 验证后端
```

---

### 模式 C: 调试/问题排查对话

**对话类型:** 修复运行时错误、优化性能、改进用户体验

**必须执行:**
1. ✅ 问题已确认解决或已记录
2. ✅ 错误日志已收集
3. ✅ 临时文件已清理
4. ✅ 解决方案已文档化
5. ✅ 下一步改进明确

**检查命令:**
```bash
npm run dev 2>&1 | tee debug.log  # 保存日志
grep -i error debug.log            # 查看错误
```

## 💾 如何保存这个检查清单

### 方式 1: 集成到VS Code任务

创建 `.vscode/tasks.json`:
```json
{
  "version": "2.0.0",
  "tasks": [
    {
      "label": "对话后检查清单",
      "type": "shell",
      "command": "echo 请执行检查清单中的步骤",
      "presentation": {
        "reveal": "always"
      }
    }
  ]
}
```

### 方式 2: 创建检查脚本

创建 `scripts/post-session-check.sh`:
```bash
#!/bin/bash
echo "=== 对话后检查清单 ==="
echo "1. 检查git状态..."
git status

echo ""
echo "2. 文档一致性检查..."
grep -r "版本:" docs/ --include="*.md"

echo ""
echo "3. 环境状态..."
echo "后端: $(curl -s http://localhost:3000 > /dev/null && echo '✅' || echo '❌')"
echo "前端: $(curl -s http://localhost:5175 > /dev/null && echo '✅' || echo '❌')"
```

运行方式：
```bash
bash scripts/post-session-check.sh
```

### 方式 3: 创建记忆库条目

在 `/memories/session/` 中保存对话总结，使用`memory`工具创建：
```
/memories/session/2026-03-10-session-summary.md
```

## 🎓 最佳实践建议

### ✅ 应该做

1. **明确文件位置** - 告诉我新文件在哪里
2. **记录变更内容** - 简要说明做了什么改变
3. **验证可运行性** - 确保代码能正常运行
4. **明确下一步** - 清楚下次应该做什么
5. **更新文档引用** - 新文件要在文档中提及

### ❌ 避免

1. ❌ 创建临时文件不清理
2. ❌ 修改文件后不保存
3. ❌ 代码有错误就结束对话
4. ❌ 没有记录改动的位置
5. ❌ 让下一次对话猜测应该做什么
